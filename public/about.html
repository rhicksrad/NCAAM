How the project came together

The Hub is a cohesive storytelling laboratory: part data pipeline, part design system, and part long-form magazine. Below is the framework that keeps every page connected.

Creative direction & authorship

Ryan Hicks defined the tone, reporting cadence, and user journeys across the site, using OpenAI Codex as a co-creator to draft copy, wrangle data, and prototype layouts.

  • Unified editorial voice across previews, history, and insights
  • Codex-assisted iteration for narrative structures and polish
  • Single-author accountability for quality and accuracy

Technical architecture

A static-first build with modular HTML, CSS, and scripted data ingestion powers fast loads and maintainable updates without heavyweight frameworks. Every data pipeline pass attempts to pull the latest numbers from the Ball Don't Lie (BDL) API as the baseline before layering manual research when gaps appear.

  • Reusable component styling in styles/
  • Dataset automation via Python notebooks and scripts
  • Accessible-by-default markup and semantic structure

Research & storytelling engine

Season previews, historical retrospectives, player dossiers, and data experiments all flow from a shared library of stats, league context, and scenario modeling.

  • Cross-era comparisons and trend analysis
  • Playstyle tagging for team and player pages
  • Scenario planners linking insights to future updates
